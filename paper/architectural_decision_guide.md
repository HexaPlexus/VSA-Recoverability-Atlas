# Architectural Decision Guide

| Scenario | Recommended representation | Recommended recovery path | Required verifier | Fallback | Main cost | Main risk | Evidence status |
| --- | --- | --- | --- | --- | --- | --- | --- |
| Known exact structure | Semantic payload + exact first-order sidecar manifest | Recursive replay with memoization | Digest / record-integrity check | Typed failure and abstention | R5 exact side information; replay latency | Wrong-but-valid or stale handles | ADOPTED_ENGINEERING_BASELINE |
| Approximate semantic lookup | MAP semantic payload + mature exact packed baseline | Exact binary scan or adopted mature index before any structure readout | Exact trace / record association check | Broader exact scan or abstain | R11 preprocessing and index bytes | Similarity != exact provenance | SUPPORTED_DEVELOPMENT_ONLY |
| Clean single-product factorization | Use the dominant native substrate for the current lawful contract | BCF native in the tested common clean F=3 envelope; MAP only as easy-cell fast path | Tuple reconstruction verifier | Exact abstain or selected stronger native method | R4 structured code or R7 decoder compute | Contract overclaim outside clean F=3 | SUPPORTED_DEVELOPMENT_ONLY |
| Unknown composite after retrieval | Ordinary sidecar DAG | Exact replay, not blind factorization | Manifest and semantic digest validation | Typed failure | R5 exact side information | Does not solve initial localization | ADOPTED_ENGINEERING_BASELINE |
| Noisy symbolic channel | Typed corruption contract + explicit abstention | Substrate-specific evaluation only; do not merge raw p across substrates | Typed protocol gates and held-out discipline | Do not claim unmatched noise frontier | Protocol and evaluation overhead | Protocol leakage | CONFIRMED_IN_FROZEN_ENVELOPE |
| Deep recursive structure | Exact sidecar DAG with memoized replay | Traverse unique reachable nodes only | Cycle / stale-parent detection | Replay budget exhaustion | Replay latency and cache state | Deep latency if uncached | SUPPORTED_DEVELOPMENT_ONLY |
| Memory-constrained workspace | Prefer exact baselines or equal-bit controls before custom compressed side channels | Compare against extra dimensions and scalar quantization first | Physical byte accounting | Adopt simpler baseline if frontier is dominated | R1 versus R2 trade-off | Storage non-dominance | SUPPORTED_DEVELOPMENT_ONLY |
| Latency-critical workspace | Use trivial static route only if it captures the measured benefit | Easy-cell MAP fast path, otherwise BCF or exact baseline | Zero silent-wrong acceptance | Always use dominant single method | Probe latency and dual-view storage | Non-cost-effective early exit | SUPPORTED_DEVELOPMENT_ONLY |
| Safety-critical decision | Verifier-first with abstention and exact fallback | Accepted output only after typed verification | Mandatory | ABSTAIN / exact fallback | Reduced coverage | Silent wrong acceptance | DESIGN_PRINCIPLE |
| Hardware-limited deployment | Consult literature-only hardware frontier separately | Do not infer hardware gains from repo CPU experiments | Comparability-class discipline | Stay with software baseline | R10 physical parallelism; hardware co-design | Speculative transfer | LITERATURE_ONLY |
