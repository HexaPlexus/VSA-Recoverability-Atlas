# Owner Review Checklist

The following decisions cannot be made automatically inside the manuscript-hardening stage and should be confirmed by the repository owner before any public preprint or public GitHub release.

## Identity and authorship

- Public author name
- Preferred public GitHub handle, if any
- Author affiliation
- Contact email for manuscript and repository
- ORCID, if desired

## Legal and release metadata

- Repository license
- Whether `Project Alexandria` appears publicly or remains an internal codename
- Preferred citation wording in `CITATION.cff`
- Whether historical `CGRN-HSR` should appear in subtitle or only in compatibility note

## Publication choices

- Preprint platform
- Target venue or venue family
- Whether the current markdown manuscript should be converted to LaTeX before external review
- Whether the systematic mapping remains part of the same paper or becomes a companion supplement

## Disclosure sections

- Acknowledgements
- Funding statement
- Conflict-of-interest statement
- Data and code availability wording

## Public repository decisions

- Whether all result artifacts currently tracked in `results/` remain public
- Whether any large rebuildable artifact should move to a release asset or archive
- Whether hardware-literature-only figures belong in the main paper or in supplement

## Final owner sign-off

- Approve claim scope as currently written
- Approve public project title: `VSA Recoverability Atlas`
- Approve manuscript title
- Approve bibliography completeness level for external circulation
