# Research History

## Historical Identity

The repository began as `CGRN-HSR`, a narrower hypothesis about context-guided recoverability in Vector Symbolic Architectures.

It is now better described as an atlas of:

- reproduced baselines,
- bounded positive results,
- explicit negative results,
- exact-structure alternatives,
- protocol repairs,
- and architectural stop conditions.

## Phase Summary

### Level 0

- Established dependency discipline and a reproducible TorchHD/MAP bootstrap.

### Level 1

- Confirmed bounded context-conditioned candidate routing gains over random subsets.
- Rejected warm continuation as a default runtime path.
- Audited BCF transfer at a scoped selector/native-envelope level.

### Level 2

- Narrowed temporal and controller seams.
- Rejected broad custom entity-resolution or metareasoning architecture claims.

### Level 3

- Measured MAP budget limits and clean-U1 transition behavior.
- Reproduced the NeCo linear-code paper contract.
- Closed algebraic baselines with a symbolic exact lower bound.
- Introduced explicit noise-contract and confirmatory gate discipline.
- Tested semantic-to-trace routing, first-order trace seams, exact capsules, exact manifests, and several encoder/soft-information alternatives.

## Repeated Pattern

Across many stages the repository converged on the same lesson:

- if a method only reshuffled the same weak evidence, it usually lost to simpler baselines;
- if a method preserved exact structure, it often worked but stopped being “blind recovery”;
- if a method added complexity without a new nondominated point, it was blocked;
- if ambiguity remained, abstention and exact verification were the honest answer.

## Public Transition

The public-release stage does not erase the historical name `CGRN-HSR`.

Instead it preserves:

- old result directories,
- old protocol IDs and hashes,
- old document names where they are part of scientific provenance,

while presenting the whole repository publicly as **VSA Recoverability Atlas**.
